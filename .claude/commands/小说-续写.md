# 小说续写/扩写/改写向导

对已有小说内容进行续写、扩写、改写或缩减。自动加载项目上下文，保持风格与角色一致性。

## 前置检查

1. 检查当前目录是否为小说项目（是否存在 `项目信息.md`）
2. 如果不存在，提示用户先执行 `/小说-新建`
3. 读取以下文件作为上下文：
   - `项目信息.md` — 基础配置
   - `世界观.md` — 世界观设定
   - `角色/` — 角色档案目录
   - `大纲.md` — 故事大纲
   - `伏笔.md` — 伏笔追踪表
   - `状态.md` — 角色状态
   - `写作风格指南.md` — 风格指南

---

## 第一步：选择操作模式

请用户选择本次操作类型：

**【选项】**
1. ✍️ **续写** — 从当前内容末尾继续往下写，保持情节连贯
2. 📝 **扩写** — 对现有段落扩充细节，增加描写深度（环境、心理、对话等）
3. 🔄 **改写** — 修改指定段落的风格、语气或内容方向
4. ✂️ **缩减** — 精简冗余内容，保留核心情节和关键信息
5. 💬 **自定义** — 用户输入具体需求（如"把这段对话改得更口语化"）

---

## 第二步：指定目标内容

根据所选模式，确认操作范围：

**【选项】**
1. 📄 处理最近一次生成的内容
2. 📍 指定段落范围（如"第3-5段"）
3. 📋 粘贴/引用具体文本片段
4. 📖 处理整个当前章节

**【提示】** 若用户未指定，默认处理最近一次生成的内容。

---

## 第三步：确认写作参数

汇总当前上下文信息，向用户确认：

| 参数 | 当前值 |
|------|--------|
| 操作模式 | （第一步选择） |
| 目标范围 | （第二步选择） |
| 字数预期 | 续写: 800-1500字 / 扩写: 原文1.5-2倍 / 改写: 与原文相当 / 缩减: 原文50%-70% |
| 视角 | 自动检测前文视角（第一/第三人称） |
| 时态 | 自动检测前文时态 |

**【选项】**
1. ✅ 确认，开始生成
2. ⚙️ 调整参数（用户可修改字数、视角、时态等）
3. ➕ 补充额外要求（如"加入更多对话"、"减少心理描写"等）

---

## 第四步：执行生成

根据确认的参数执行写作：

1. 分析前文风格特征（句式长短、修辞偏好、节奏感）
2. 提取角色当前状态（情绪、位置、关系变化）
3. 对照大纲确认情节走向不偏离主线
4. 生成内容，保持与前文无缝衔接

**【提示】** 生成过程中自动遵循：
- 角色性格与已建立的人设一致
- 用词风格与前文统一（古风/现代/文艺等）
- 不引入前文未提及的设定
- 伏笔与悬念按大纲节奏安排

---

## 第五步：结果确认与迭代

生成完成后展示内容，并提供后续选项：

**【选项】**
1. ✅ 满意，写入文件
2. 🔄 再来一版（相同参数重新生成）
3. 🔧 微调（指定具体修改点，如"对话再生动些"）
4. 🔀 切换模式（如从续写切换到扩写处理刚生成的内容）
5. ❌ 放弃本次生成

**【提示】** 用户选择"写入文件"后，自动追加到对应章节文件末尾（续写）或替换原文（扩写/改写/缩减）。
