# 经验教训管理向导

记录和管理小说创作过程中的经验教训，帮助提高写作质量。

## 前置检查

1. 检查当前目录是否为小说项目（是否存在 `项目信息.md`）
2. 如果不存在，提示用户先执行 `/小说-新建`

---

## 第一步：操作类型选择

**【选项】**
1. 📝 **记录经验** — 记录一条新的经验教训
2. 📋 **查看经验** — 查看所有经验教训
3. 🔍 **搜索经验** — 按关键词搜索经验
4. 📊 **经验统计** — 查看经验统计数据
5. 📤 **导出经验** — 导出经验教训文档

---

## 第二步：记录经验（如选择记录）

### 经验类型
**【选项】**
1. ✅ **成功经验** — 做得好的地方，值得复用
2. ❌ **失败教训** — 做得不好的地方，需要避免
3. 💡 **改进建议** — 可以改进的地方
4. ⚠️ **注意事项** — 需要注意的事项
5. 📝 自定义类型

### 经验分类
**【选项】**
1. 📖 **写作技巧** — 关于写作本身的经验
2. 👤 **角色塑造** — 关于角色设计的经验
3. 🌍 **世界观设定** — 关于世界观的经验
4. 🎯 **伏笔管理** — 关于伏笔的经验
5. ⏰ **节奏控制** — 关于剧情节奏的经验
6. 🔍 **一致性检查** — 关于检查的经验
7. 📝 自定义分类

### 经验描述
**【提示】** 用户输入经验的具体内容：
- 问题描述：发生了什么问题？
- 原因分析：为什么会发生？
- 解决方案：怎么解决的？
- 经验总结：以后怎么避免？

### 关联章节
**【选项】**
1. 📍 指定关联章节

## 工作流指引

### 使用场景
| 场景 | 推荐操作 |
|------|----------|
| 写完一章后，记录经验 | 记录经验 |
| 遇到问题时，查找解决方案 | 搜索经验 |
| 定期总结，查看经验统计 | 经验统计 |
| 项目结束后，导出经验 | 导出经验 |

### 相关技能
- `/小说-章节` — 写作时可以记录经验
- `/小说-检查` — 检查时可以发现问题并记录
- `/小说-润色` — 润色时可以记录改进经验

### 经验管理最佳实践
1. **及时记录**：遇到问题或成功时立即记录
2. **定期总结**：每10章总结一次经验教训
3. **分类管理**：按类型和分类组织经验
4. **应用经验**：在后续写作中应用经验教训

## 项目经验参考

### 《星际流浪者》项目经验（2026年6月2日）

#### 成功经验
1. **世界观先行**：先建立完整的世界观，再开始写作
2. **角色设定要立体**：主角要有明确的性格特点和弱点
3. **大纲规划要细致**：大纲要分卷、分章、分场景
4. **伏笔管理要严格**：伏笔要提前规划，不能临时起意
5. **写作风格要统一**：写作风格要根据小说类型选择

#### 失败教训
1. **大纲与实际写作不一致**：大纲要预留调整空间
2. **角色发展速度过快**：角色的成长要有合理的节奏
3. **反派塑造不够深入**：反派要有合理的背景和动机
4. **感情线发展太快**：感情线要有合理的节奏
5. **伏笔回收不够自然**：伏笔的回收要自然

#### 改进建议
1. **大纲管理**：大纲要更详细，要预留调整空间
2. **角色管理**：角色档案要更详细，状态要及时更新
3. **伏笔管理**：伏笔要提前规划，要及时回收
4. **写作流程**：写作流程要标准化，要可追溯
5. **工具优化**：写作技能要更完善，要更智能
