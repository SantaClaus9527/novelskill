# 章节撰写向导

按章节撰写小说正文，自动加载项目上下文保持一致性。

## 前置检查

1. 检查当前目录是否为小说项目（是否存在 `项目信息.md`）
2. 如果不存在，提示用户先执行 `/小说-新建`
3. 读取以下文件作为上下文：
   - `项目信息.md` — 基础配置
   - `世界观.md` — 世界观设定
   - `角色/` — 角色档案目录
   - `大纲.md` — 故事大纲
   - `伏笔.md` — 伏笔追踪表
   - `状态.md` — 角色状态
   - `写作风格指南.md` — 风格指南

---

## 第一步：章节选择

### 选择写哪一章
**【选项】**
1. 📖 写下一章（自动检测当前进度）
2. 📍 指定章节号
3. 🔄 重写某章（覆盖已有章节）
4. 📝 从大纲选择章节

**【提示】** 如果是第一章，会自动初始化章节开头。

---

## 第二步：章节规划确认

根据大纲，显示本章规划信息：

| 项目 | 内容 |
|------|------|
| 章节号 | 第X章 |
| 章节名 | {大纲中的章节名} |
| 章节目标 | {本章要达成什么} |
| 主要场景 | {大纲中的场景} |
| 涉及角色 | {本章出场角色} |
| 伏笔任务 | {本章需埋设/回收的伏笔} |
| 预估字数 | 2500-5000字 |

**【选项】**
1. ✅ 确认，按大纲写
2. ✏️ 调整章节规划
3. ➕ 补充额外要求

---

## 第三步：场景设计

### 场景数量
**【选项】**
1. 🎬 1个场景（整章一个连续场景）
2. 🎭 2-3个场景（推荐）
3. 📍 4+个场景（快节奏）
4. 🤖 AI根据内容自动安排

### 场景转换方式
**【选项】**
1. ⏭️ 直接切换（场景间空行）
2. ⏳ 时间跳跃（"三天后..."）
3. 🚶 空间移动（角色移动到新地点）
4. 💭 回忆/闪回
5. 📝 自定义过渡

---

## 第四步：写作参数确认

| 参数 | 当前值 |
|------|--------|
| 视角 | 从项目信息读取 |
| 时态 | 过去时（默认）/ 现在时 |
| 风格 | 从写作风格指南读取 |
| 字数 | 2500-5000字 |

**【选项】**
1. ✅ 使用默认参数
2. ⚙️ 调整参数

---

## 第五步：正文生成

AI 根据以上设定生成章节正文：

**生成策略：**
1. 先生成场景框架（每个场景的核心事件）
2. 填充对话和动作
3. 添加环境描写和心理描写
4. 检查角色一致性
5. 嵌入伏笔（如有）

**【选项】**
1. 📄 一次性生成整章
2. 🎬 逐场景生成（每写完一个场景暂停确认）
3. 📝 先写关键对话，再补充描写

---

## 第六步：章节检查

生成完成后自动检查：

1. ✅ 角色性格是否一致
2. ✅ 世界观是否符合
3. ✅ 与前文是否连贯
4. ✅ 伏笔是否正确埋设/回收
5. ✅ 字数是否达标

**检查结果：**
- ✅ 通过 — 无问题
- ⚠️ 警告 — 有小问题但可接受
- ❌ 错误 — 需要修改

---

## 第七步：结果确认

**【选项】**
1. ✅ 满意，保存章节
2. 🔄 重新生成整章
3. 🔧 局部修改（指定段落重写）
4. ➕ 添加内容（补充场景/对话/描写）
5. ✂️ 删除内容（精简冗余部分）

---

## 执行动作

1. 创建/更新 `章节/第{XXX}章.md` 文件
2. 更新 `状态.md`（角色状态变化、时间线推进）
3. 更新 `伏笔.md`（如有伏笔变动）
4. 更新 `项目信息.md` 中的进度信息

**章节文件结构：**
```markdown
# 第X章：{章节名}

{正文内容}

---
**本章要点：**
- 主要事件：{简述}
- 角色变化：{简述}
- 伏笔埋设：{如有}
- 伏笔回收：{如有}
```

---

## 输出完成信息

1. 章节字数统计
2. 本章关键事件
3. 下一章预告（根据大纲）
4. 进度更新（已完成X章/共Y章）
