# 写作技能进化

> **别名**：`/小说-升级`、`/小说-迭代`
>
> 基于鉴赏家反馈和写作经验，自动升级整套写作技能系统。这是一个**元技能**——它改进的是技能本身，而非小说内容。
>
> **核心理念**：写作技能不应该是静态的，它应该随着你写的越多、反馈越多而越来越精准。

---

## 触发条件

本技能通常由以下场景触发：

| 触发来源 | 场景 | 输入 |
|----------|------|------|
| `/小说-鉴赏家` | 鉴赏报告发现了技能包未覆盖的写作技法 | 鉴赏报告全文 |
| `/小说-经验` | 积累了足够多的同类型经验教训 | 经验教训汇总 |
| 用户主动 | 用户觉得现有技能不够用，想针对性升级 | 用户描述的需求 |
| 卷末回顾 | 写完一卷后，回顾整体表现 | 本卷所有鉴赏报告+经验记录 |

---

## 第一步：进化输入确认

**【选项】**
1. 🧬 **从鉴赏报告进化** — 读取最近的鉴赏报告，提取可升级的点
2. 📋 **从经验教训进化** — 读取 `经验教训.md`，汇总高频问题
3. 📝 **用户指定进化方向** — 用户直接描述想增强的能力
4. 🔄 **全面体检** — 对照已完成的所有章节和鉴赏报告，全面扫描技能包的盲区

---

## 第二步：技能包诊断

AI 对照以下清单，分析当前技能包的覆盖情况和盲区：

### 诊断维度

| 维度 | 检查项 | 对应技能文件 |
|------|--------|------------|
| **叙事技法** | 展示vs告知、视角运用、时间线操控、信息释放节奏 | `小说-润色`、`小说-检查` |
| **人物塑造** | 心理深度、对话个性化、关系动态、成长弧设计 | `小说-人设`、`小说-检查` |
| **情节工程** | 张力曲线、伏笔密度、转折设计、节奏控制 | `小说-大纲`、`小说-章节`、`小说-伏笔` |
| **文笔打磨** | 感官描写、比喻系统、句式变化、语言风格一致性 | `小说-润色`、`小说-章节` |
| **世界观构建** | 设定自洽、细节丰富度、规则一致性 | `小说-世界观`、`小说-检查` |
| **类型要素** | 该类型小说的核心爽点/泪点/悬念是否到位 | `小说-检查`、`小说-章节` |
| **长篇管理** | 角色漂移、伏笔遗忘、时间线混乱、节奏失衡 | `小说-检查`、`小说-状态`、`小说-经验` |

### 诊断输出格式

```markdown
## 🔍 技能包诊断报告

### ✅ 覆盖良好的领域
- {领域}：{简述为什么做得好}

### ⚠️ 覆盖不足的领域
- {领域}：{具体缺失什么，来源于哪些鉴赏反馈/经验}

### ❌ 完全空白的领域
- {领域}：{需要新增什么能力}
```

---

## 第三步：进化方案设计

基于诊断结果，AI 设计具体的进化方案：

### 进化类型

| 类型 | 说明 | 操作 |
|------|------|------|
| 📝 **补充规则** | 在现有技能中新增检查/润色规则 | 编辑已有 `.md` 文件 |
| 🔧 **优化流程** | 调整技能的执行步骤或选项 | 编辑已有 `.md` 文件 |
| ➕ **新增维度** | 在检查/润色/鉴赏中新增评估维度 | 编辑已有 `.md` 文件 |
| 🆕 **新增技能** | 创建全新的技能文件 | 创建新 `.md` 文件 |
| 📊 **调整参数** | 修改默认值、阈值、比例建议 | 编辑已有 `.md` 文件 |
| 🔗 **加强关联** | 在技能之间建立更强的工作流连接 | 编辑多个 `.md` 文件 |

### 进化方案格式

```markdown
## 🧬 进化方案

### 发现的问题
{来源于鉴赏报告/经验教训的具体问题}

### 当前技能包的缺口
{哪个技能文件缺少了什么}

### 进化操作
| 序号 | 操作类型 | 目标文件 | 具体改动 | 预期效果 |
|------|---------|---------|---------|---------|
| 1 | 📝 补充规则 | `小说-润色.md` | 新增XX规则 | 解决XX问题 |
| 2 | ➕ 新增维度 | `小说-检查.md` | 新增XX检查项 | 避免XX问题 |

### 改动预览
{展示每个文件的关键改动内容}
```

**【选项】**
1. ✅ **全部接受** — 执行所有进化操作
2. 📝 **选择性接受** — 选择执行哪些操作
3. 🔧 **调整方案** — 修改进化方案后再执行
4. ❌ **放弃** — 不执行进化

---

## 第四步：执行进化

根据确认的方案，AI 执行进化操作：

1. 编辑目标技能文件（备份原内容到改动记录中）
2. 确保改动不破坏现有功能
3. 保持技能文件之间的交叉引用一致性
4. 更新 `README.md` 中的技能说明（如有新增技能）

### 安全约束
- **不删除已有功能**：只增不减，除非用户明确要求
- **保持向后兼容**：新增的选项/维度不应影响默认流程
- **改动可追溯**：记录每次进化的内容和原因

---

## 第五步：进化记录

每次进化后，自动记录到进化日志：

```markdown
# 技能进化日志

## {日期} — 进化 #{序号}

**触发来源**：{鉴赏报告/经验教训/用户指定/卷末回顾}

**进化内容**：
- {改动1}
- {改动2}

**影响的文件**：
- `小说-XXX.md`：{改动概述}

**预期效果**：{解决什么问题}

**验证方式**：{下次写作时如何验证进化是否生效}
```

进化日志保存到 `经验教训.md` 的专门区域，或单独的 `技能进化日志.md`。

---

## 第六步：进化验证建议

进化完成后，提示用户如何验证效果：

**【提示】**
```
🧬 技能进化完成！下次写作时可以通过以下方式验证：
1. 写新章节时，观察进化后的技能是否自动覆盖了之前的盲区
2. 用 `/小说-鉴赏家` 对新章节做快速点评，看同类问题是否减少
3. 写完5-10章后，做一次全面鉴赏对比进化前后的整体质量
```

## 工作流指引

### 典型使用流程
```
日常写作循环：
  章节写作 → 鉴赏家点评 → 修改 → 满意 → 记录经验
                ↓
        发现系统性问题（不是个案，是技能包的盲区）
                ↓
          /小说-进化 → 诊断 → 方案 → 执行
                ↓
          技能包升级完成 → 回到日常写作循环
```

### 建议触发频率
- **日常**：不触发，除非鉴赏报告明确指出"当前技能包未覆盖"
- **每卷结束**：建议做一次全面体检，根据本卷积累的鉴赏反馈和经验进化
- **里程碑**：每50章或每2卷做一次深度进化

### 与其他技能的关系
- **输入**：`/小说-鉴赏家`（鉴赏报告）、`/小说-经验`（经验教训）、用户反馈
- **输出**：修改 `小说-润色`、`小说-检查`、`小说-章节`、`小说-人设` 等所有技能文件
- **闭环**：写作 → 鉴赏 → 发现问题 → 进化技能 → 更好的写作 → 更高质量的鉴赏
