# 小说项目创建向导

创建一个新的小说项目，初始化完整的目录结构和基础文件。

## 交互流程

### 第一步：小说类型选择

请用户选择小说类型（给出选项，用户可输入补充）：

**【选项】**
1. 🌸 言情（霸道总裁/甜宠/虐恋/校园恋爱）
2. ⚔️ 玄幻/修仙（修仙/玄幻/奇幻/异世界）
3. 🔍 悬疑/推理（悬疑推理/刑侦/本格/社会派）
4. 🚀 科幻（末日/赛博朋克/太空歌剧/硬科幻）
5. 📜 历史（架空历史/古代/穿越重生）
6. 🏙️ 都市（都市异能/职场/商战）
7. 🎮 其他（用户自定义输入）

**【提示】** 如果用户选择"其他"，请询问具体类型描述。

---

### 第二步：篇幅规模选择

**【选项】**
1. 📖 短篇（5万字以内，1-20章）
2. 📚 中篇（5-20万字，20-80章）
3. 📕 长篇（20-50万字，80-200章）
4. 📚📚 超长篇（50万字+，200章+，适合连载）

---

### 第三步：叙事视角选择

**【选项】**
1. 👤 第一人称（沉浸感强，适合言情/悬疑）
2. 👥 第三人称限知（跟随主角视角，最常用）
3. 🌐 第三人称全知（上帝视角，适合群像/史诗）
4. 🔄 多视角切换（不同章节不同视角，适合悬疑/群像）

---

### 第四步：写作风格选择

**【选项】**
1. 🎨 文艺细腻（注重意境、情感描写，适合言情/文艺）
2. ⚡ 爽文节奏（快节奏、爽点密集，适合网文/玄幻）
3. 🧊 冷峻写实（克制、注重逻辑细节，适合悬疑/科幻）
4. 🎭 智能适配（根据类型自动切换，推荐）

---

### 第五步：小说名称输入

请用户输入小说名称（自由输入）。

**【提示】** 如果用户没有想好，可以提供几个风格化建议供选择。

---

### 第六步：一句话简介

请用户用一句话描述小说核心卖点（自由输入）。

**【示例格式】**
- "一个废柴少年偶得上古传承，从此踏上逆天修仙之路"
- "高冷总裁爱上平凡女主，却不知她是他失散多年的青梅竹马"
- "连环杀手在警探眼皮底下作案，每次留下一个谜题"

---

## 执行动作

根据用户选择，创建以下目录结构和文件：

```
{小说名称}/
├── 项目信息.md          # 基础配置（类型/篇幅/视角/风格）
├── 世界观.md            # 世界观设定（待填充）
├── 角色/
│   └── .gitkeep
├── 大纲.md              # 分层大纲（待填充）
├── 伏笔.md              # 伏笔追踪表
├── 状态.md              # 角色状态/时间线
├── 写作风格指南.md      # 风格指南（根据选择生成）
└── 章节/
    └── .gitkeep
```

**项目信息.md** 包含：
```yaml
---
小说名称: {名称}
类型: {类型}
篇幅: {规模}
视角: {视角}
风格: {风格}
一句话简介: {简介}
创建时间: {当前日期}
当前进度: 0章/{预计章数}
---
```

**写作风格指南.md** 根据选择生成对应的风格指导：
- 文艺细腻：强调意象运用、情感层次、环境烘托
- 爽文节奏：强调爽点设置、节奏控制、升级打怪
- 冷峻写实：强调细节真实、逻辑严谨、氛围营造
- 智能适配：根据类型自动匹配风格要素

---

## 输出完成信息

创建完成后，输出：
1. 项目目录结构
2. 下一步建议（推荐先执行 `/小说-世界观` 设定世界观）
3. 项目文件路径
